# 股票年度已实现盈利计算器

为了应对 CRS 监管并辅助报税，本项目通过 **富途 OpenD API** 自动获取股票成交记录，计算每个年度的已实现盈利，作为报税和复盘依据。

## 支持平台

| 平台 | 状态 |
|------|------|
| 富途牛牛 | ✅ 已支持（仅成交数据） |

> **注意**：目前仅支持通过富途 API 获取成交记录，不包含新股申购等非交易数据，需要手工补充后再计算。

## 功能特性

- 通过富途 OpenD API 自动获取成交记录
- 手工补充新股申购等缺失数据后合并
- 支持多种成本核算方法（AVERAGE / FIFO / LIFO / HIFO / LOFO）
- 按年度、按币种统计已实现收益
- 支持期权交易和到期处理
- 支持做空交易的收益计算
- 提供 Web 可视化界面和命令行工具

## 完整使用流程

```
┌───────────────┐     ┌───────────────────┐     ┌────────────────┐     ┌──────────────────┐
│ 1. web_ui.py  │     │ 2. 手工补充新股   │     │ 3. 合并数据    │     │ 4. 计算盈利/税费 │
│ 拉取富途成交  │────▶│ 申购等缺失数据    │────▶│ merge_new_stock│────▶│ calculate_profit │
│ → futu_trade  │     │ → new_stock.csv   │     │ → futu_trade   │     │ _web.py          │
└───────────────┘     └───────────────────┘     └────────────────┘     └──────────────────┘
```

### 步骤 1：拉取富途成交数据

通过 `web_ui.py` 的 Web 界面从富途 OpenD 下载成交记录：

```bash
streamlit run web_ui.py
```

访问 `http://localhost:8501`，选择时间范围后点击 **"开始下载富途数据"**，成交记录将保存到 `.cache_data/futu_trade.csv`。

> 富途 API 只能获取普通买卖成交记录，**新股申购（暗盘/公开发售）** 等数据不在其中，需要在下一步手工补充。

### 步骤 2：手工补充新股申购数据

创建 `.cache_data/new_stock.csv` 文件，格式与 `futu_trade.csv` 一致。必填字段如下：

| 字段 | 说明 | 示例 |
|------|------|------|
| `code` | 股票代码 | `HK.09888` |
| `stock_name` | 股票名称 | `百度集团-SW` |
| `deal_market` | 市场 | `HK` 或 `US` |
| `qty` | 数量 | `100` |
| `price` | 成交价格 | `252.00` |
| `trd_side` | 交易方向 | `BUY` |
| `create_time` | 成交时间 | `2024-03-28 09:00:00` |
| `fee_amount` | 手续费（无则填 0） | `50.56` |

### 步骤 3：合并数据

运行合并脚本，将新股申购数据合并到主交易文件：

```bash
python merge_new_stock.py
```

该脚本会：
- 自动备份原始 `futu_trade.csv`
- 将 `new_stock.csv` 的记录追加合并到 `futu_trade.csv`

### 步骤 4：计算盈利和税费

合并完成后，使用 `calculate_profit_web.py`（Web 界面）或 `calculate_profit_from_file.py`（命令行）进行计算：

**Web 界面（推荐）：**

```bash
streamlit run calculate_profit_web.py
```

在界面中选择数据文件、成本核算方法和日期范围，点击 **"开始计算"** 即可查看按币种和年份分组的已实现收益。

**命令行：**

```bash
python calculate_profit_from_file.py .cache_data/futu_trade.csv --method FIFO --start-date 2024-01-01 --end-date 2024-12-31
```

## 成本核算方法

| 方法 | 说明 | 税务适用性 |
|------|------|-----------|
| **移动加权平均法** | 将所有买入成本加权平均 | 默认方法，国际通用 |
| **FIFO** | 先进先出法，先买入的先卖出 | 美国 IRS 默认方法 |
| **LIFO** | 后进先出法，后买入的先卖出 | 仅作参考 |
| **HIFO** | 最高成本优先法 | 税务筹划参考 |
| **LOFO** | 最低成本优先法 | 税务筹划参考 |

### 各国税法要求

| 地区 | 推荐方法 |
|------|---------|
| 美国税务居民 | FIFO 或 Specific Identification |
| CRS 报税 | 移动加权平均法 |

## 成本核算算法说明

### 移动加权平均法（默认）

将所有买入成本加权平均，计算公式：

```text
真实买入价 = 买入价格 + 费用/数量
真实卖出价 = 卖出价格 - 费用/数量
持仓均价 = 总成本 / 总数量
已实现收益 = 平仓数量 × (真实卖出价 - 持仓均价)
```

### FIFO（先进先出法）

假设先买入的股票先卖出，按买入顺序确定成本。

### LIFO（后进先出法）

假设后买入的股票先卖出，按买入逆序确定成本。

### HIFO（最高成本优先法）

假设成本最高的股票先卖出，可最大化成本、最小化利润。

### LOFO（最低成本优先法）

假设成本最低的股票先卖出，最小化成本、最大化利润。

## 注意事项

1. **数据完整性**：富途 API 仅提供普通买卖成交记录，新股申购、股票拆合并等非常规交易需要手工补充后再计算，否则持仓和盈亏会不准确。
2. **税务合规**：不同国家/地区对成本核算方法有不同要求，请根据您的税务居民身份选择合适的方法。
3. **数据安全**：请妥善保护交易数据与本地缓存文件，不要将敏感数据提交到版本控制系统。
4. **期权处理**：系统会自动检测并处理过期期权。
5. **做空支持**：支持做空交易的收益计算。
